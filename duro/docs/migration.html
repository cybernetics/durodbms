Migrating data from older versions

Currently Duro contains no tool that migrates data to a newer version. So the only safe way to migrate data to a newer version is to perform the following steps:

  1. Create a new database environment
  2. Create the database(s)
  3. Create the user-defined types and operators
  4. Create the real tables and import the data from the old database environment
  5. Create the virtual tables
  6. Create the database constraints

Durodump can be used to extract the real tables and their data from the old DB environment and to import it into the new environment. You may want to take durodump.tcl from the new version and use it under the old version to export the real tables because there are some issues with durodump.tcl in older versions.
