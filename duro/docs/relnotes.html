Release Notes for Duro 0.6

Functions not implemented

The following functions are not implemented in Duro 0.6:

  • RDB_gdivide
  • RDB_group
  • RDB_ungroup
  • RDB_unwrap
  • RDB_wrap
  • RDB_sdivide
  • RDB_project_tuple
  • RDB_join_tuples
  • RDB_unwrap_tuple
  • RDB_wrap_tuple
  • RDB_obj_tuple
  • RDB_set_obj_tuple
  • RDB_array_set_tuple
  • RDB_set_array_length
  • RDB_is_empty
  • RDB_create_constraint
  • RDB_drop_constraint
  • RDB_begin_uio
  • RDB_end_uio

Functions only partially implemented

RDB_create_table

Only scalar attribute types are supported.

It is not enforced that a candidate key may not be a true subset of another.

System-generated key values are not supported.

RDB_set_table_name

RDB_set_table_name is not supported for persistent tables.

RDB_make_persistent

RDB_make_persistent is not supported for real tables.

RDB_insert

RDB_insert is not supported for virtual tables which are the result of a MINUS, project, SUMMARIZE, or RENAME.

RDB_update

RDB_update is not supported for virtual tables.

RDB_delete

RDB_delete is not supported for virtual tables which are the result of a select, JOIN, EXTEND, project, SUMMARIZE, or RENAME .

RDB_summarize

COUNTD, SUMD, and AVGD are not supported.

RDB_table_to_array

An RDB_array created by RDB_table_to_array becomes invalid after the end of the transaction in which it was created.

The table from which a RDB_array was created must not be dropped before the array is destroyed.

RDB_implement_type

Nonscalar types are not supported as actual representations.

System-provided getter, setter, and selector functions are not supported if the possible representation has more than one component.

RDB_create_ro_op

Only scalar argument and return types are supported.

RDB_create_update_op

Only scalar argument types are supported.

Implementation-specific details

Creating libraries

Libraries for user-defined types and operators must be created using GNU Libtool. The name of the library is passed without the file extension.

RDB_create_env

The path passed to RDB_create_env is a Berkeley DB database environment directory. The directory must exist and should be empty.

RDB_open_env

The path passed to RDB_open_env is a Berkeley DB database environment directory.
